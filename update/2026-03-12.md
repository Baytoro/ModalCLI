# 2026-03-12

## What Changed
- Added new workload `workloads/tensor_core` for Tensor Core throughput benchmarking:
  - `tensor_core.cpp` / `tensor_core_base.cu` (WMMA `m16n16k16`, FP16 input + FP32 accumulate).
  - `data.py` with configurable benchmark parameters (`warps_per_block`, `num_blocks`, `kernel_iters`, `repeat`, `accumulators`).
  - `message.py` to print per-variant Tensor TFLOPS.
  - `config.json` defaulting to `benchmark` mode and full variant discovery.
- Unified benchmark variant output format in CLI:
  - Changed `custom_ms=... ms` to `... ms`.
  - Added message normalization for `key=value` into `value key` (e.g. `tflops=32.93` -> `32.93 tflops`).
- Fixed Tensor Core workload runtime build/link issue:
  - Replaced host-side `warpSize` usage with compile-time constant (`kWarpSize=32`) in `tensor_core_base.cu`.
- Renamed workload directory from `worloads/` to `workloads/`.
- Updated CLI target parsing:
  - Supports shorthand workload launch: `python cli.py <workload_name>` (e.g. `python cli.py vector_add`).
  - Keeps `-p/--path` compatible.
  - Adds fallback lookup for legacy `worloads/<name>` paths.

## Why
- Needed a dedicated workload to measure Tensor Core compute throughput (scheme A: custom WMMA kernel) with minimal memory bottleneck impact.

## Result
- Workload is ready for `python cli.py workloads/tensor_core`.
- Output includes timing plus estimated Tensor TFLOPS based on WMMA operation count.
- Verified on Modal: workload runs successfully and prints benchmark in the new compact format.
- Verified on Modal: `python cli.py vector_add` and `python cli.py tensor_core` both run successfully.
