# 2026-03-11

## 改了什么
- 运行框架重构：
  - 统一入口改为 `scripts/run.py`
  - 默认配置迁移为 `scripts/settings.json`
  - workload 结构统一为 `data.py` + `ref.py` + `<workload>.cpp` + `<workload>_*.cu`
- CLI 行为调整：
  - 默认安静输出，`--verbose` 打开详细日志
  - 结果输出改为结构化展示（按 `mode` 自适应）
  - 支持自动发现并批量测试多个 CUDA 变体
  - 支持从 workload `config.json` 用 `variants` 选择测试子集
  - 远端函数支持配置 `cpu`（当前默认 8）
- 测试模式新增：
  - `mode=accuracy`：只测正确性（需要 `ref.py`）
  - `mode=benchmark`：只测性能（不需要 `ref.py`）
  - 不写或 `mode=all`：正确性与性能都测
- 向量加法示例重构：
  - 变体改名为 `vector_add_base.cu` / `vector_add_fp4.cu`
  - `fp4` 版本改为 `float*` 接口，kernel 内手动 `float4` 访存
  - 统一改为 float32 路线（`vector_add.cpp` + `data.py` 同步约束）
  - 计时逻辑改为每 iter 使用 CUDA Event 累加，iter 间清 L2（128MB 刷写）

## 为什么改
- 提升原型迭代效率：统一入口、统一结构、减少每个 workload 的重复样板。
- 支持同一 workload 多 kernel 版本横向对比，便于快速筛选最优实现。
- 让正确性验证和性能测试解耦，避免无关步骤拖慢迭代。
- 输出更直观，便于快速判断“是否通过 + 哪个版本更快”。

## 结果
- `python cli.py -p worloads/vector_add` 多次实跑通过。
- `variants=["fp4"]` 时仅执行 `fp4` 变体。
- `mode=benchmark` 时在移除 `ref.py` 的情况下仍可正常运行。
