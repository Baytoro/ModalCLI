# 2026-03-11

## What Changed
- Refactored runner flow to shared entry `scripts/run.py` and default settings file `scripts/settings.json`.
- Added multi-variant CUDA auto-discovery (`<workload>_*.cu`) and variant filtering via workload `config.json` field `variants`.
- Added mode control in workload `config.json`:
  - `accuracy` (requires `ref.py`)
  - `benchmark` (no `ref.py` required)
  - missing / `all` (run both)
- Improved CLI output formatting:
  - structured sections
  - mode-aware variant display
  - section name changed from `Data` to `Settings`
- Added optional workload `message.py` hook:
  - `variant_message(ctx, variant, settings)`
  - if missing, no extra variant message is printed.
- Added GPU override support in workload `config.json`:
  - `gpu` overrides `scripts/settings.json` default GPU.
- Reworked `workloads/vector_add`:
  - `vector_add_base.cu` and `vector_add_float4.cu`
  - float32-only path
  - float4 variant uses `float*` interface and converts to `float4` internally.
- Added new workload `workloads/cuda_core`:
  - register-heavy FP32 FMA microbenchmark kernel
  - benchmark mode by default
  - optional TFLOPS message via `message.py`.

## Why
- Faster iteration for CUDA prototype testing.
- Clear separation of correctness vs performance workflows.
- Better support for comparing multiple kernel variants in one workload.
- Cleaner, more actionable terminal output.

## Result
- `vector_add` and `cuda_core` workloads run successfully on remote GPU.
- `benchmark` mode works without `ref.py`.
- Variant-scoped output works as expected.
